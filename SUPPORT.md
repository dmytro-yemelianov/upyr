# Support

Upyr is maintained as an open-source project without a guaranteed response time.
The following routes help keep requests actionable.

## Setup and usage questions

Use [GitHub Discussions](https://github.com/dmytro-yemelianov/upyr/discussions) for
installation, permissions, configuration, shortcuts, and general questions. Check
the README and existing discussions first.

## Bugs and model behavior

Use the repository's structured issue forms:

- **Bug report** for application, CLI, installer, or platform failures;
- **Correction or language-model report** for false corrections, missed
  corrections, capitalization, punctuation, or physical-key mapping;
- **Feature request** for proposed behavior.

Include `upyr --version`, the operating system and architecture, installation
source, clear reproduction steps, and `upyr doctor` output when relevant. Use only
synthetic example text. Redact usernames, local paths, window content, clipboard
content, and anything typed in another application.

## Security

Do not use Discussions or public issues for vulnerabilities. Follow
[SECURITY.md](SECURITY.md) and use GitHub's private vulnerability-reporting form.
